# Accessibility Baseline (WCAG 2.1 AA)

Treat accessibility as a release blocker.

## Requirements

- Semantic HTML for structure and controls.
- Complete keyboard navigation for interactive elements.
- Visible focus states.
- Form controls with labels and clear error mapping.
- ARIA only when semantic HTML cannot express the behavior.

## Review Checklist

- Heading hierarchy is meaningful.
- Color contrast is acceptable.
- Interactive targets are usable on touch devices.
- No keyboard traps or hidden focus.
